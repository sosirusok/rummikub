# 🀄 루미큐브 · 4인 실시간 (Rummikub)

브라우저에서 4명이 실시간으로 즐기는 루미큐브. 백엔드 서버 없이 **정적 사이트 + Supabase Realtime**으로 동작합니다.

## 동작 방식
- 접속하면 **1·2·3·4** 중 자리를 고릅니다. 한 번호는 한 명만 선택 가능(좌석 테이블 기본키로 중복 차단).
- **4명이 다 차면 자동으로 게임 시작.**
- 타일을 손패에서 보드로 드래그(마우스/터치 모두 지원)해 세트를 만들고 `턴 종료`.
- 못 내거나 안 낼 땐 `한 장 뽑기`.

## 규칙 (정식 루미큐브)
- 타일 106장: 1~13 × 4색 × 2벌 + 조커 2장. 각자 14장으로 시작.
- **그룹**: 같은 숫자, 서로 다른 색, 3~4장.
- **런**: 같은 색 연속 숫자 3장 이상.
- **첫 등록**: 처음 내려놓을 땐 새로 만든 세트 합이 **30점 이상**이어야 하며, 이때는 보드의 기존 세트를 건드릴 수 없음.
- 첫 등록 이후엔 보드를 자유롭게 재배열하며 타일을 추가 가능(단, 매 턴 최소 1장은 새로 내려놓아야 함).
- 손패를 가장 먼저 모두 내려놓는 사람이 승리.
- *조커 회수(보드의 조커를 실제 타일로 교체해 가져오기)는 현재 버전 미지원.*

## 설정 (최초 1회)
1. **Supabase**: `supabase_setup.sql` 내용을 대시보드 → SQL Editor 에 붙여넣고 RUN.
2. **config.js**: `SUPABASE_URL` 과 `SUPABASE_ANON_KEY` 를 채움 (Project Settings → API).
   - anon 키는 클라이언트 노출용이라 공개 레포에 넣어도 안전합니다.
3. **배포**: GitHub Pages (Settings → Pages → Branch: `main` / root).

## 파일
| 파일 | 역할 |
|---|---|
| `index.html` | 진입점 |
| `styles.css` | 스타일(펠트 테이블/타일/트레이) |
| `config.js` | Supabase 접속 정보 |
| `engine.js` | 룰 엔진(타일·세트 검증·턴 검증) |
| `net.js` | Supabase Realtime 동기화 |
| `app.js` | UI·드래그&드롭·게임 진행 |
| `supabase_setup.sql` | DB 1회 설정 스크립트 |

## 새 게임 / 초기화
종료 화면의 `새 게임` 또는 로비의 `방 초기화` 버튼으로 좌석을 비우고 다시 시작합니다.
