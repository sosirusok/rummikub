# 🀄 루미큐브 · 실시간 (v2)

모바일/노트북 브라우저에서 친구들과 즐기는 실시간 루미큐브. **로그인 → 방(1~5) → 좌석/관전 → 방장이 시작 → 게임 → 누적 랭킹**.

## 주요 기능
- **계정/로그인**: 게스트 불가. 아이디·비밀번호·본명(한글)으로 가입, 아이디·비밀번호로 로그인. 비밀번호는 DB에서 bcrypt 해시.
- **방 5개 로비**: 각 방 `(착석인원)/4` 표시. 방 입장 후 좌석 선택 또는 **관전**.
- **방장**: 빈 방에 처음 들어가면 방장. 방장이 **턴 제한시간(15/30/60초)**을 고르고 **2~4명**일 때 시작.
- **관전**: 관전자는 플레이어를 탭해 그 사람 손패를 봄. 플레이어끼리는 서로 손패 비공개(화면).
- **모바일 탭 조작**: 손패를 **탭해서 고르고**(여러 장 가능) 보드/＋새 세트를 **탭해서 놓기**. 드래그 불필요. 숫자/색 정렬, 되돌리기/리셋, 30점 진행바, 세트 유효성 색표시.
- **턴 타이머**: 서버 시각 기준 카운트다운(시계오차 보정). 시간 초과 시 자동 뽑기. 끊긴 플레이어는 다른 클라가 대리 진행.
- **누적 점수/전적**: 좌석마다 본명·전적(N승M패)·누적점수. 랭킹 화면.

## 점수 규칙 (누적, 음수 불가 / floor 0)
- 기본: 2인 `[+60, -20]`, 3인 `[+80, +20, -40]`, 4인 `[+100, +40, 0, -60]` (1등→꼴찌 순).
- 순위 = 남은 손패 점수 오름차순(빈 손=1위). **동점은 차지한 자리들의 평균** 점수.
  - 예) 4인 1·2·2·4 → 두 2등 각 **+20**. 4인 1·2·3·3 → 두 3등 각 **-30**. 3인 1·3·3 → 두 동점 각 **-10**.
- 승 = 1등. 점수는 서버 함수 `rk_finish_game`가 보드 state로 직접 산정(클라가 점수를 못 위조).

## 설정 (최초 1회)
1. **Supabase SQL**: `supabase_setup.sql` 전체를 대시보드 → SQL Editor 에 붙여넣고 RUN.
   - `users`(RPC 전용), `rooms`(5개), `room_members` 생성 + RPC + realtime. (v1의 room/seats 는 자동 삭제.)
   - 이메일 인증 토글 불필요 — 커스텀 계정이라 가입 즉시 로그인.
2. **config.js**: `SUPABASE_URL` / `SUPABASE_ANON_KEY` (Project Settings → API).
3. **배포**: GitHub Pages (Settings → Pages → main / root).

## 알려진 범위 (친구용 캐주얼)
- 게임 보드 상태와 좌석은 친구 신뢰 기반(상대 손패 비공개는 화면 차원). 누적 **점수**는 서버에서 덱 무결성(타일 보존·산더미 상한)을 검증해 위조를 막지만, 완전한 서버측 수(手) 검증은 범위 밖.
- 단일 글로벌 방 1~5(룸코드 없음). 조커 회수(보드 조커 교체)는 미지원.
